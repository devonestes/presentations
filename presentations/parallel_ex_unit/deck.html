You can think of the code server as a big key-value store, where keys are function names and values are bytecode for functions. This part of the code server is called the `global export table`. --- class: center, middle ![](/assets/images/cs2.jpg) ??? For the most part (of course there are exceptions), the code server deals with modules. You can load or unload a module from the code server, and the code server actually keeps _two_ versions of a module around if you load a new one!
